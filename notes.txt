Process/step based ui,

2. Upload requirements and click validate - Request gets sent to back end to validate the mapping against the schema. If sucess the request gets stored in the front end 
3. Upload test cases and click validate - Request gets sent to back end to validate the mapping against the schema. 
3.5 If both uploads pass the second request gets combined with the first to create a payload 
4. If both uploads have passed and then the user clicks run the combined payload is sent to the back end and executes the engine 
5. Once ran the data is returned and presented in the front end
